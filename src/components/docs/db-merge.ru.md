# DB_MERGE — обзор

*EN version: [DbMergeDocs.astro](https://github.com/nikitak-dev/nikitak-dev.github.io/blob/main/src/components/docs/DbMergeDocs.astro)*

---

### // WHAT IT DOES

Даёт единый источник правды, который не нужно собирать вручную: одну таблицу, что остаётся актуальной, не содержит дубликатов и читается одинаково везде, куда смотрит команда.

### // HOW IT WORKS

Каждый источник подтягивается по расписанию, его несовпадающие поля приводятся к единой форме, затем обогащаются общими справочными данными. Каждая запись получает устойчивый ключ, и этот ключ управляет upsert (вставкой или обновлением): каждый запуск обновляет уже существующие строки, а не добавляет новые копии. Если источник вернул пусто, запуск останавливается и поднимает флаг, а не перезаписывает корректные данные пустотой.

### // GOOD TO KNOW

Серверная автоматизация — управлять здесь нечем. Демонстрация работает на синтетических банковских записях и держит Google Sheet и базу Airtable синхронными, отправляя короткую сводку в командный чат после каждого запуска.
